## 背景

随着技术积累增加，个人最大的挑战不是获取知识，而是如何让已有经验长期保存、快速恢复并持续复用。

过去一年中积累了大量工程经验：

- RV1106 AI视觉开发
    
- RKMPI媒体框架
    
- RKNN部署优化
    
- STM32底层开发
    
- FOC电机控制
    
- ODrive电机驱动
    
- 二轮平衡车建模与控制
    
- 状态空间、LQR、MPC等控制理论
    

但是这些知识大量存在于：

- 大脑记忆
    
- ChatGPT聊天记录
    
- 临时笔记
    
- 项目代码
    

没有形成稳定的长期知识资产。

本质问题：

> 知识仍然停留在RAM中，没有固化到非易失存储。

---

# 个人知识状态模型

## RAM：实时工作记忆

特点：

- 当前项目上下文完整
    
- 理解最深
    
- 调试经验丰富
    
- 问题定位速度快
    

例如：

正在开发RV1106时：

```
VI
 |
VPSS
 |
RKNN
 |
VENC
```

整个架构、代码、问题都在脑中。

但是项目结束后：

- 上下文消失
    
- 细节遗忘
    
- 再次启动需要重新学习
    

---

## Cache：近期经验

项目过程中形成：

- 调试记录
    
- 问题分析
    
- 解决方案
    
- 关键代码片段
    

例如：

```
问题：
OpenCV resize导致RV1106性能下降

原因：
CPU参与图像格式转换

方案：
使用VPSS硬件处理

结果：
FPS提升
```

几个月内仍然有效。

---

## SSD：长期工程知识

最终目标：

形成个人工程知识库。

保存：

- 原理
    
- 架构
    
- 设计决策
    
- 踩坑经验
    
- 方法论
    

例如：

从RV1106项目抽象出的经验：

> 嵌入式视觉系统中，应优先使用硬件pipeline减少CPU参与和数据搬运。

这个知识已经超越RV1106，可以迁移到其他平台。

# 为什么普通笔记不够

传统笔记：

```
机器人/
 ├── 平衡车.md

控制/
 ├── LQR.md

嵌入式/
 ├── RV1106.md
```

问题：

- 文件之间关系弱
    
- 时间久后不知道为什么记录
    
- 无法形成知识体系
    

真正需要的是：

```
知识网络
```

例如：

```
二轮平衡车

 ├── 倒立摆模型
 |
 ├── 状态空间
 |
 ├── LQR控制
 |
 ├── IMU融合
 |
 ├── 电机FOC
 |
 └── ODrive
```

- Debug过程

重点：
