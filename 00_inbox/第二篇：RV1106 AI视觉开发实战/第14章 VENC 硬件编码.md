# 二、为什么必须有 VENC？

如果没有 VENC，你只能得到：

```
YUV / RGB 图像帧
```

问题非常严重：

- 单帧 1080p ≈ 6MB
- 30fps ≈ 180MB/s
- 网络无法传输
- 存储无法承受

------

## 👉 结论

> 视频必须压缩成“码流”，而不是原始图像

------

# 三、VENC 是什么？

VENC（Video Encoder）是：

> **硬件视频编码模块**

在 Rockchip RV1106 中，支持：

- H.264（主力）
- MJPEG（部分场景）
- H.265（部分平台）

# 五、VENC vs CPU编码

## 5.1 CPU编码（不可用方案）

```
YUV → x264 → CPU压缩
```

问题：

- CPU占用极高
- 实时性差
- 功耗大

------

## 5.2 VENC硬件编码（推荐）

```
YUV → VENC → H.264（硬件）
```

特点：

- 零CPU编码压力
- 实时 1080p@30fps
- 工业级稳定

------

# 六、H.264 编码基础（必须理解）

------

## 6.1 GOP结构

```
I帧 → P帧 → P帧 → P帧 → I帧
```

- I帧：关键帧（完整图像）
- P帧：差分帧
- B帧：双向预测（部分系统支持）

------

## 6.2 为什么要 GOP？

- 减少带宽
- 提高压缩率
- 支持快进/恢复

------

# 八、VENC核心流程（非常重要）

------

## Step 1：创建编码通道

```
create_venc_channel();
```

------

## Step 2：设置编码参数

```
venc_attr.codec = H264;
venc_attr.width = 1920;
venc_attr.height = 1080;
venc_attr.fps = 30;
venc_attr.bitrate = 4000; // kbps
```

------

## Step 3：绑定 VPSS

```
VPSS → VENC
```

------

## Step 4：启动编码

```
start_venc();
```

------

## Step 5：取码流

```
get_stream(&stream);
```

------

# 九、码流结构（重点）

VENC 输出不是“图片”，而是：

```
NALU 单元
```

包含：

- SPS（参数集）
- PPS（图像参数）
- IDR（关键帧）
- P帧数据

------

# 十、关键参数解析

------

## 10.1 Bitrate（码率）

```
越高 → 清晰度越高 → 带宽越大
```

推荐：

- 720p：2000 kbps
- 1080p：4000~8000 kbps

------

## 10.2 FPS

必须与 VI / VPSS 对齐：

```
30fps → 全链路30fps
```

------

## 10.3 GOP

常见：

- 30（1秒1I帧）
- 60（2秒1I帧）

------
